Theoretical Part

1. Ethernet standards for physical layer, ethernet limits, Ethernet frame

Ethernet standards for physical layer - speed versions. Transmission speed measured in millions of bits per second (Mbps) under ideal conditions:

  • Standard - 10 Mbps
  • Fast - 100 Mbps
  • Gigabit - 1000 Mbps
  • 10 Gigabit - 10 Gbps

Symbols - XXBaseXX

  • First number is the speed of the network, eg. 100 for Fast Ethernet
  • "Base" stands for "baseband" - transmissions carry one signal at a time (in opposite to broadband)
  • The end tail can indicate several things. Number is max. length of the cable in hundred meters, eg. 5 => 500 meters. Letters indicate the type of the cable. (T - twisted pair, SX, LX, SR, LR, ER, FX... - several types of fiber optic

Ethernet frame - data packet format for transmitting data over an Ethernet network.

Preamble - 7 bytes and Start Frame Delimiter - 1 byte
A sequence of alternating bits 7 times (101010100) and one (10101011) used to synchronize the transmitter and receiver before the actual frame begins.

Destination Address - 6 bytes and Source Address - 6 bytes
MAC addresses identifying the intended receiver and the sender. In modern switched networks, frames are forwarded only to the correct port; broadcast and multicast frames are delivered to all hosts.

Type Field - 2 bytes
Indicates the upper-layer protocol contained in the payload. Common values include IPv4, IPv6, and ARP; older protocols such as IPX and AppleTalk may also appear.

CRC - 4 bytes
A 32-bit cyclic redundancy check used for error detection. If the CRC does not match, the receiver discards the frame without requesting retransmission.

2. MAC adress, broadcasting

MAC adress - unique adress hard-wired into every network device by manufacturer. MAC addresses are unique; no two network devices made by any manufacturer anywhere in the world can have the same MAC address.

Broadcasting - one-to-all communication method. It allows single sender to transmits a data packet to every device on a specific network or broadcast domain.

3. The difference between hub, switch and router

Hub is multiport repeater. It regenerates and amplifies every signal received on any port on all of it's other ports. Hubs do not examine the contents of the signal that they regenerate and that is why they are working at Physical Layer. Works in half-duplex - it cannot send and receive at the same time.

Switch connects devices within a LAN. It is an intelligent hub that examines the MAC address of arriving frames to determine which port to forward it (unicast). It supports full-duplex - it can send and recive at the same time.

Router connects different networks (different IP subnets), such as LAN and the internet and forwards data packets between them using IP addresses.

4. CSMA/CD, collision, half and full duplex

CSMA/CD - Carrier Sense Multiple Access/Collision Detection
Whenever a device wants to send a packet over the network media, it first listens to the network media to see whether anyone else is already sending a packet. If it doesn't hear any other signals on the media, the computer assumes that the network is free, so it sends the packet. Nothing prevents two or more devices from trying to send a message at the same time. After a device sends a packet, it listens carefully to see whether the packet crashes into another packet. If it crashes, it waits a random period of time and then tries to send the packet again. Because the delay is random, two packets that collide are sent again after different delay periods, so a second collision is unlikely.

Collision - occurs when more than one node transmits. In bus topology it can be detected by comparing transmitted data with received or by recognizing higher amplitude of the signal than usual. If signal sent is different than received back, the node sends jam signal which notifies collision on the network.

Half-duplex - Data flows in both directions, but only in one direction at a time. Devices take turns sending and receiving data. The available bandwidth is shared, so the effective throughput is lower.

Full-duplex - Data flows in both directions at the same time. Dedicated bandwidth for sending and receiving, leading to higher throughput and no collision issues.

5. ARP request, detecting doubled IP address

When a device needs to send data to another device on the same network, it first checks its internal ARP cache (a table of IP-to-MAC address mappings) for the destination IP address. If the IP address is not in the cache, the device broadcasts an ARP request on the local network segment. The request contains the IP and MAC address of the sender, and the IP address of the intended recipient. The recipient's MAC address field is set to a broadcast address to ensure all devices receive it. Every device on the local network receives the broadcast.

Detecting doubled IP address - duplicate IP address issue occurs when more than one station share the same address. ARP is able to detect such a situation. Firstly, it can prevent the conflict with probe (Who has 192.168.1.15?). Secondly, if two stations have the same address, it is visible via ARP protocol that they share the same address (192.168.1.15 it's AA:AA and in the same time 192.168.1.15 it's BB:BB).

6. BOOTP, DHCP

DHCP - Dynamic Host Configuration Protoco; allows individual computers on a TCP/IP network to obtain thei IP address and other communication parameters from a server. The DHCP server keeps track of which IP addresses are already assigned so that when a computer requests an IP address, the DHCP server offers it an IP address that is not already in use.

BOOTP - uses a static method for IP address allocation. The IP address is assigned based on the MAC address of the client, meaning that the same IP address is assigned to a client eachtime it requests one.

7. VLANs

VLANs - allows multiple separate logical networks to exist on the same physical LAN infrastructure. VLAN tagging is implemented according to the 802.1Q specification, which inserts a VLAN tag inside the Ethernet frame. Devices in different VLANs are isolated from each other. Communication between VLANs requires a Layer 3 device (router or Layer 3 switch). VLANs are ideal for separating groups of devices that should not communicate directly, improving security, reducing broadcast traffic, and organizing the network.
